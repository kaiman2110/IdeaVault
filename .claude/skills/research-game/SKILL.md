参考ゲーム・プロダクトの設計を **WebSearch + WebFetch** で徹底調査し、自プロジェクトへの適用案を出す。

引数 = $ARGUMENTS

## フェーズ 1: 調査対象の特定

1. $ARGUMENTS を解析し、以下を判断:
   - **名前指定**の場合 → その製品全体 or focus 領域を調査
   - **テーマ指定**の場合 → 複数の成功事例を横断調査
2. プロジェクトの現在の設計を把握する（CLAUDE.md、設計ドキュメント）
3. 関連する既存メモリ・アイデアを確認する

## フェーズ 2: Agent teams で並列リサーチ

**3〜4 の Agent を同時起動**し、異なる角度から調査する。

### テーマ指定の場合
```
Agent 1: 代表的な成功事例を調査
Agent 2: 設計論・ベストプラクティスを調査
Agent 3: ユーザー評価・コミュニティ反応を調査
```

### 名前指定の場合
```
Agent 1: mechanics — コアメカニクスの設計思想
Agent 2: economy — 経済・リソース設計
Agent 3: ux — UI/UX・フィードバック設計
Agent 4: narrative — ストーリーテリング・世界観表現
```

## フェーズ 3: 結果統合 + 自プロジェクトへの適用

全 Agent の結果を統合し、以下のレポートを作成:

```markdown
# リサーチレポート: {テーマ or 名前}

## 調査サマリー
## 調査結果

## 自プロジェクトへの適用分析

### 取り入れるべき要素（推奨）
### 参考にすべきだが慎重に検討する要素
### 見送るべき要素
```

## フェーズ 4: 成果の保存

1. レポートを `docs/research/` に保存
2. 「取り入れるべき要素」から具体的なアイデアを `docs/ideas.md` に追記
3. 「`/design` に渡して具体的な仕様設計に進みますか？」と確認
