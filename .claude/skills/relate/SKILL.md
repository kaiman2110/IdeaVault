description: Vault 全体をスキャンしてメモ間の関連性を分析し、related フィールドに Obsidian リンクを追加する。
argument-hint: "[対象メモのファイル名（省略時は Vault 全体）]"
---

メモ間の関連付けを行う。引数 = $ARGUMENTS

## フェーズ 1: メモの収集

1. $ARGUMENTS が指定されている場合 → そのメモを中心に関連を探す
2. 指定なしの場合 → Vault 全体（inbox/, incubating/, actionable/, projects/, references/）をスキャン
3. 各メモの frontmatter（tags, type, needs）と本文を読み込む
4. archive/ は除外（完了・取り下げ済み）

## フェーズ 2: 関連性の分析

以下の基準で関連性を判定:

| 基準 | 重み | 例 |
|------|------|-----|
| **タグの一致** | 高 | 両方に `godot` タグ |
| **type の一致** | 中 | 両方 `idea` |
| **本文のキーワード一致** | 中 | 同じ技術名・概念が登場 |
| **project の一致** | 高 | 同じプロジェクトに紐づく |
| **参照の連鎖** | 低 | A が参照する記事と B が同じトピック |

## フェーズ 3: 関連の提案

分析結果をユーザーに提示:

```
## 関連付けの提案

| メモ A | メモ B | 関連度 | 理由 |
|--------|--------|--------|------|
| セーブ機能の改善案 | Godot リソース管理 | 高 | タグ: godot, 内容: Resource 関連 |
| シェーダー記事 | ピクセルアート研究 | 中 | タグ: godot, shader |

適用しますか？（yes / 個別選択 / no）
```

## フェーズ 4: related フィールドの更新

ユーザー承認後:
1. 各メモの frontmatter `related` フィールドに Obsidian リンクを追加
   - 形式: `"[[ファイル名]]"`（拡張子なし）
   - **既存のリンクは削除しない** — 新規リンクのみ追加
2. **双方向リンク**: A → B を追加したら B → A も追加
3. `updated` を今日の日付に更新

## フェーズ 5: コミット

- メッセージ例: `relate: メモ3件の関連付けを更新`
- Vault コンテンツ操作なので main 直接コミット可

## 注意事項

- **既存の related リンクは削除しない**（追加のみ）
- **frontmatter の id, created は変更しない**
- 関連度が低いものは提案に含めない（ノイズ防止）
- 自分自身へのリンクは追加しない
