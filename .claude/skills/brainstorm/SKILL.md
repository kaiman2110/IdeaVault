メモをブラッシュアップする。引数 = $ARGUMENTS

## フェーズ 1: 対象メモの特定

1. $ARGUMENTS がファイルパスの場合 → そのファイルを読む
2. ファイル名のみの場合 → `inbox/`, `incubating/` から検索
3. 引数なしの場合 → `needs: brainstorm` のメモ一覧を表示し選択を促す

## フェーズ 2: 内容の読み込み・分析

1. メモの frontmatter と本文を読み込む
2. 現在の内容を分析:
   - 何についてのアイディアか
   - どこまで具体化されているか
   - 不足している観点は何か

## フェーズ 3: ブラッシュアップ

メモの末尾に以下を**追記**する（元の内容は変更しない）:

### 追記する内容
1. **アイディアの構造化**: 散文を整理し、要点を明確化
2. **具体化**: 曖昧な部分を具体的なステップや仕様に落とし込む
3. **補足**: 関連する技術・手法・参考情報を追記
4. **疑問点・検討事項**: 次に考えるべきポイントをリストアップ
5. **次のアクション提案**: research / design / execution のどれが適切か

### 追記フォーマット
```markdown

---

## ブラッシュアップ（YYYY-MM-DD）

### 要点の整理
（構造化した内容）

### 具体化
（具体的なステップや仕様）

### 関連情報
（技術・手法・参考）

### 検討事項
- （疑問点や未解決の課題）

### 次のアクション
- [ ] （提案するアクション）
```

## フェーズ 4: Frontmatter 更新

1. `updated` を今日の日付に更新
2. `needs` を内容に応じて更新:
   - まだ調査が必要 → `research`
   - 設計に進める → `design`
   - そのまま brainstorm 継続 → 変更なし
3. `tags` に新たに判明したタグを追加（既存は削除しない）

## フェーズ 5: コミット

- メッセージ例: `brainstorm: セーブ機能の改善案をブラッシュアップ`
- Vault コンテンツ操作なので main 直接コミット可
- 現在 feature ブランチにいる場合はそのブランチでコミット

## 注意事項

- **メモの内容を勝手に削除しない**（追記のみ）
- **frontmatter の id, created は変更しない**
- ユーザーの元の文章を書き換えない — 構造化は別セクションで追記
