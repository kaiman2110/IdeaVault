セッションを終了する。以下を**すべて自動実行**すること。

> **重要**: ユーザーが「終わり」「おわり」「ここまで」「exit」「quit」「終了」等と言ったら、
> **必ずこのスキルを実行してからセッションを終了する**。
> `/exit` コマンドはセッションを即座に閉じるため、メモリが保存されない。
> ユーザーが exit の意思を示したら、先にこのスキルを実行し、完了後に
> 「セッション終了準備が完了しました。`/exit` で終了できます」と案内する。

## 1. 未コミット変更の確認

```
git status --short
git diff --stat
```

未コミットの変更がある場合:
- ユーザーに「コミットしますか？」と確認
- コミットする場合は動作確認項目を提示してから実施

## 2. 並列ドキュメント更新（Agent teams）

以下を **3つの Agent を同時起動** して並列更新する。
**必ず1メッセージで3つの Agent tool を並列呼び出しすること。**

```
Agent(general-purpose) ×3 並列:
  ├─ Agent 1: MEMORY.md + Issue テーブル更新
  │    「gh issue list --state open で最新状態を取得し、
  │     MEMORY.md の Issue テーブルを更新。
  │     完了 Issue → ✅ Done、新規 Issue → 追加、MS進捗を反映。」
  │
  ├─ Agent 2: CLAUDE.md + ロードマップ更新
  │    「セッション中に以下があったか確認し、あれば更新:
  │     - CLAUDE.md: 新アーキ原則 / 落とし穴 / スキル / Hook
  │     - docs/roadmap.md: MS完了 / 先送り / 新アイデア / 更新履歴」
  │
  └─ Agent 3: セッション知見の保存
       「セッション中に得られた知見をメモリファイルに保存:
        - 新しいバグパターン・落とし穴
        - ユーザーからのフィードバック・修正指示
        - プロジェクト方針の変更・決定
        保存不要と判断した場合は '保存対象なし' と報告。」
```

## 3. 引継ぎサマリー

並列更新の完了後、次のセッションで参照できるよう以下をまとめて表示:
- 今回の作業内容の要約
- 残作業・次にやるべきこと
- ブロッカーや注意点
- 現在のブランチと状態
- 保存したメモリ一覧（ある場合）
