Agent teams を使ってプロジェクトを**並列に多角解析**する。引数 = $ARGUMENTS（デフォルト: all）

## all の場合: 4エージェントを同時起動

以下の4つの Agent を **1つのメッセージで同時に** 起動する:

### Agent 1: 構造分析（structure）
```
プロンプト: このプロジェクトの構造を分析してください。
1. グローバル状態・シングルトンの一覧と依存関係
2. シーン・ページ遷移のグラフ
3. フォルダ構成とファイル数・行数の統計
4. 最も変更頻度の高いファイルTop10（git log から）
日本語で結果をまとめてください。
```

### Agent 2: コード品質分析（quality）
```
プロンプト: このプロジェクトのコード品質を分析してください。
1. God Class の検出: 大きなファイルを列挙し、責務分割の提案
2. 重複コードの検出: 類似パターンが3箇所以上に散在しているもの
3. 型ヒント不足のファイル
4. CLAUDE.md の原則・落とし穴に該当するパターンがないか
5. refactoring.md の項目が解消済みかチェック
日本語で結果をまとめてください。
```

### Agent 3: シグナル・通信分析（signals）
```
プロンプト: このプロジェクトのイベント・シグナル通信を分析してください。
1. グローバルイベントバスの全シグナル/イベント定義
2. 各イベントの発火箇所と購読箇所を全検索
3. 未使用イベント（定義あるが使われていない）
4. 未購読イベント（発火されるが誰もリスンしていない）
5. イベントバスを経由せず直接参照しているケース
日本語で結果をまとめてください。
```

### Agent 4: コンテンツ整合性（content）
```
プロンプト: このプロジェクトのデータ・コンテンツ整合性を検証してください。
（プロジェクト固有のチェック — CLAUDE.md のデータ構造を参照して検証内容を判断）
日本語で結果をまとめてください。
```

## 個別 focus の場合

$ARGUMENTS で指定された1つの分析のみ Agent 1つで実行する。

## 結果統合

全エージェントの結果が返ったら、以下の形式で統合レポートを作成:

```markdown
# プロジェクト分析レポート（YYYY-MM-DD）

## サマリー
- 検出された問題: X件（高: X / 中: X / 低: X）
- 新たなリファクタリング候補: X件

## 構造分析
（Agent 1 の結果）

## コード品質
（Agent 2 の結果）

## シグナル・通信
（Agent 3 の結果）

## コンテンツ整合性
（Agent 4 の結果）

## アクション項目
| 優先度 | 項目 | 対象ファイル | 推奨対応 |
|--------|------|-------------|---------|
| ...    | ...  | ...         | ...     |
```
