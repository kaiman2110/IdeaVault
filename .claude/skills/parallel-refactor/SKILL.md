Agent teams と worktree 分離を使って**複数のリファクタリングを安全に並列実行**する。

> **安全性**: 各エージェントは `isolation: "worktree"` で独立したコピーで作業する。
> 変更が衝突しないことを確認してからメインブランチに統合する。

## フェーズ 1: リファクタリング項目の選定

1. `refactoring.md`（memory 内）を読み、現在の項目一覧を取得
2. $ARGUMENTS が `all` の場合 → 全項目を対象
3. $ARGUMENTS が項目名の場合 → 指定された項目のみ対象
4. 各項目の**変更対象ファイルを特定**し、衝突チェックを行う

### 衝突チェック
- 2つ以上の項目が**同じファイルを変更**する場合 → **並列不可**（順次実行にフォールバック）
- 変更ファイルが完全に独立 → **並列実行OK**

## フェーズ 2: 並列実行計画をユーザーに提示

```markdown
## リファクタリング実行計画

### 並列グループ 1（同時実行）
- [ ] 項目A: ... → Agent 1 (worktree)
- [ ] 項目C: ... → Agent 2 (worktree)

### 順次グループ（グループ1完了後）
- [ ] 項目B: ... → 項目Aと変更ファイルが重複

変更ファイルの衝突はありません。実行しますか？
```

ユーザーの承認を待つ。

## フェーズ 3: Agent teams で並列実行

承認後、並列可能な項目を **Agent tool で同時起動** する。

各エージェントの設定:
- `subagent_type: "general-purpose"`
- `isolation: "worktree"` — 独立した作業コピーで安全に変更
- プロンプトに以下を含める:
  - リファクタリングの具体的な内容
  - 変更対象ファイルのリスト
  - CLAUDE.md のアーキテクチャ原則
  - テスト実行コマンド
  - コミットメッセージは日本語で

## フェーズ 4: 結果統合

1. **worktree の変更確認**: 各エージェントの worktree パスとブランチ名を記録
2. **テスト結果確認**: 全エージェントがテスト通過しているか
3. **変更内容レビュー**: 各 worktree の diff をセルフレビュー

### 統合手順（全エージェント成功時）
各 worktree のブランチを順次メインブランチにマージ:
```bash
git merge <worktree-branch> --no-edit
# テスト実行（マージ後の整合性確認）
```

### 統合手順（一部エージェント失敗時）
- 成功した worktree のみマージ
- 失敗した項目を報告し、手動対応を提案

## フェーズ 5: 後処理

1. `refactoring.md` から完了した項目を削除
2. 新たに発見した技術的負債があれば追記
3. 結果サマリーをユーザーに報告
