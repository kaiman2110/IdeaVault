現在の変更をプロジェクトのアーキテクチャ原則に照らしてレビューする。

1. `git diff` で変更内容を確認する（ステージ済み+未ステージ両方）
2. 変更されたソースファイルを読む
3. **CLAUDE.md を読み、アーキテクチャ原則と既知の落とし穴をチェックリストとして使う**
   - CLAUDE.md の「アーキテクチャ原則」セクションの各項目を確認
   - CLAUDE.md の「既知の落とし穴」テーブルのパターンが変更コードに含まれていないか確認
4. 問題があれば具体的な修正提案を出す
5. 問題がなければ「レビュー通過」と報告する
